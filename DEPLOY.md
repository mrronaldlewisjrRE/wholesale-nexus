# Deploying WholesaleNexus

Follow this guide to deploy your Real Estate Wholesaling Platform using **Supabase** (Backend) and **Vercel** (Frontend).

## Prerequisites

- [Node.js](https://nodejs.org/) installed
- A [GitHub](https://github.com/) account
- A [Vercel](https://vercel.com/) account
- A [Supabase](https://supabase.com/) account

---

## Part 1: Supabase Setup (Database)

1.  **Create a Project**:
    - Log in to Supabase and create a new project named `WholesaleNexus`.
    - Set a strong database password.

2.  **Create the Database Table**:
    - Go to the **SQL Editor** in the Supabase dashboard.
    - Run the following SQL to create the `deals` table:
    ```sql
    create table deals (
      id bigint generated by default as identity primary key,
      created_at timestamp with time zone default timezone('utc'::text, now()) not null,
      address text not null,
      stage text not null, -- e.g., 'Under Contract', 'Prospecting'
      value text,          -- e.g., '$12,000'
      date text,           -- e.g., 'Oct 12'
      status text          -- e.g., 'active', 'closed', 'pending'
    );
    ```

3.  **Insert Sample Data**:
    - Still in the SQL Editor, add some initial data:
    ```sql
    insert into deals (address, stage, value, date, status) values
    ('123 Oak Lane', 'Under Contract', '$12,000', 'Oct 12', 'active'),
    ('442 Elm Drive', 'Prospecting', '$8,500', 'Nov 01', 'pending'),
    ('8592 Pine Street', 'Closed', '$15,200', 'Sep 28', 'closed');
    ```

4.  **Get API Keys**:
    - Go to **Project Settings** -> **API**.
    - Copy the `Project URL` and `anon public` key. You will need these for Vercel.

---

## Part 2: GitHub Setup

Since you are running locally, you need to push this code to GitHub.

1.  Initialize Git (if not done):
    ```bash
    git init
    git add .
    git commit -m "Initial commit"
    ```
2.  Create a new repository on GitHub.
3.  Push your code:
    ```bash
    git remote add origin https://github.com/YOUR_USERNAME/wholesale-nexus.git
    git branch -M main
    git push -u origin main
    ```

---

## Part 3: Vercel Deployment (Frontend)

1.  **Import Project**:
    - Log in to Vercel and click **Add New...** -> **Project**.
    - specific the GitHub repository you just created (`wholesale-nexus`).

2.  **Configure Environment Variables**:
    - In the "Environment Variables" section of the import step, add the Supabase keys you copied earlier:
    
    | Name | Value |
    |------|-------|
    | `VITE_SUPABASE_URL` | Your Supabase Project URL |
    | `VITE_SUPABASE_ANON_KEY` | Your Supabase anon public key |

3.  **Deploy**:
    - Click **Deploy**.
    - Vercel will build your React app.
    - Once finished, you will get a live URL (e.g., `https://wholesale-nexus.vercel.app`).

---

## Part 4: Local Development (Optional)

To connect your local environment to Supabase:

1.  Create a file named `.env` in the root directory.
2.  Add your keys:
    ```
    VITE_SUPABASE_URL=your_project_url
    VITE_SUPABASE_ANON_KEY=your_anon_key
    ```
3.  Install the Supabase client library:
    ```bash
    npm install @supabase/supabase-js
    ```
4.  Restart your dev server:
    ```bash
    npm run dev
    ```
